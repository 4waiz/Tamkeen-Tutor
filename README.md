# SkillCompass UAE

**AI-powered skills-gap analyzer and adaptive tutor for UAE youth.**
Built for the Tamkeen 5.0 “AI for Education & Skills” challenge.

SkillCompass diagnoses a learner’s current skill level, identifies weak areas,
creates a personalized 7-day learning path, teaches with an AI tutor, and tracks
measurable progress — all with a bold neobrutalism UI.

> **Find your weakest skills. Get a 7-day plan. Improve with guided practice.**

---

## ✨ What it does

1. **Sign up / sign in** — real Supabase email + password auth with protected routes.
2. **Onboarding** — name, age range, language (English / Arabic / bilingual), learning goal, confidence (1–5), accessibility preference. Persisted to Supabase.
3. **Diagnostic assessment** — 8 multiple-choice questions mapped to skill categories for the chosen goal. Scored server-side; answers and per-skill scores saved.
4. **Skill-gap report** — overall readiness, every skill category scored, two weakest skills, recommended next lesson, progress trend, and a clear action plan.
5. **Personalized learning path** — a 7-day plan generated by **AI** (when a key is configured) or a **deterministic local fallback**. Persisted as path + daily lesson items.
6. **AI tutor** — chat that knows your goal, language, skill scores, and current lesson; gives UAE-relevant examples and one check-for-understanding question. Server-side only; falls back gracefully.
7. **Practice & progress** — submit practice work per lesson, get AI or rubric feedback, mark lessons complete, watch the dashboard update.
8. **Mentor dashboard** — cohort view: all learners, average readiness, most common weak skills, learners at risk (<50%), recent completions.

Everything works **without an AI key** — every AI route has a tested fallback.

## 🧱 Tech stack

- **Next.js 14** (App Router) + **TypeScript**
- **Tailwind CSS** — neobrutalism design tokens from `SKILL.md`
- **Supabase** — auth + Postgres + Row Level Security (`@supabase/ssr`)
- **OpenAI-compatible AI** — via a server-side route (any Chat Completions endpoint)
- **Zod** — validation for every form and AI output
- **React Hook Form** — auth + onboarding forms
- Pure **SVG/CSS** charts (no chart library, no paid UI kits)

## 🚀 Local setup

### 1. Prerequisites
- Node.js 18.18+ (tested on Node 24)
- A free [Supabase](https://supabase.com) project

### 2. Install
```bash
npm install
```

### 3. Configure environment
```bash
cp .env.example .env.local
```
Fill in at minimum the two **public** Supabase values (see `.env.example`):
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`

AI keys are **optional** — skip them to run in fallback mode.

### 4. Set up the database
In the Supabase dashboard → **SQL Editor** → **New query**, paste the contents of
[`supabase/schema.sql`](supabase/schema.sql) and run it. This creates all tables,
Row Level Security policies, and a trigger that auto-creates a profile row on signup.

### 5. (Recommended for the demo) Disable email confirmation
Supabase → **Authentication → Providers → Email** → turn **off** “Confirm email”.
This lets new sign-ups land straight in onboarding. With it on, users must confirm
via email before signing in (the app handles both paths).

### 6. Run
```bash
npm run dev
```
Open <http://localhost:3000>.

## 🤖 AI provider configuration

The app calls an **OpenAI-compatible Chat Completions API** from server routes only —
keys are never exposed to the browser. Configure in `.env.local`:

```env
AI_API_KEY=sk-...                 # or OPENAI_API_KEY
AI_BASE_URL=https://api.openai.com/v1
AI_MODEL=gpt-4o-mini
```

Works with OpenAI, OpenRouter, Together, local LM servers, etc. — anything that
implements `/chat/completions`.

**No key?** No problem. `/api/generate-learning-path`, `/api/tutor`, and
`/api/evaluate-practice` each detect the missing key and use the deterministic
fallback generators in `src/lib/content/`. The app is fully functional either way.

## 🗄️ Database tables

`profiles`, `assessments`, `assessment_answers`, `skill_scores`, `learning_paths`,
`learning_path_items`, `tutor_messages`, `lesson_submissions`, `lesson_completions`.

All user-owned tables have RLS policies restricting rows to `auth.uid()`. The mentor
view uses the optional **service-role key** server-side to aggregate across learners;
without it, the mentor view safely shows only the signed-in user.

## 🎨 Design system

Implements the `SKILL.md` neobrutalism system:
- Tokens in `tailwind.config.ts` (primary `#FDC800`, secondary `#432DD7`, success/warning/danger, warm `#FBFBF9` surface, `#1C293C` ink).
- Typographic scale 13/15/17/21/27/35; Inter (display) + JetBrains Mono (labels/code).
- Thick borders, offset hard shadows, pressed-active states.
- Every interactive component has default / hover / focus-visible / active / disabled / loading / error states.
- WCAG 2.2 AA: visible focus ring, labelled fields, `aria-*` wiring, skip link, keyboard navigation, reduced-motion support.
